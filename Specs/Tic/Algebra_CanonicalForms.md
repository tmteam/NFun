# Канонические формы — Optional

## Зачем канонические формы — и почему это уровень алгебры

Optional-семантика имеет два представления: **конструктор** `opt(τ)`
(StateOptional) и **флаг** `[D..A]?` (IsOptional на CS). Без нормализации одна и та
же семантика существует в двух формах одновременно, и каждый Apply/Merge обязан
обрабатывать обе — правила дублируются и расходятся.

Канонические формы — **инвариант алгебры**, а не стадий: каждый оператор
(∨, ∧, ⊓, ↓, ↑, simp) обязан и *принимать*, и *возвращать* только канонические
состояния. Именно на этот инвариант опирается каждый Apply/Merge в Pull/Push — они
никогда не проверяют «а вдруг пришла неканоническая форма».

## Rule B — Optional-лифт нерешённых границ

> **Rule B**: `opt(τ)` требует **решённого** τ. Optional-лифт нерешённого
> ограничения `[D..A]` — это **flag form** `[D..A]?`, а не `opt([D..A])`.

```
lift([D..A])   = [D..A]?          (нерешённое — флаг)
lift(T)        = opt(T)           (решённое — конструктор)
lift(Any)      = Any              (Opt(Any) = Any, см. ниже)
```

Обоснование — закон refinability (§ниже): состояние `opt([D..A])` содержит
нерешённый CS **внутри конструктора**, вне узла графа. Никакое ребро не достигает
его, никакой Pull/Push не уточнит — это «мёртвый остров», замороженный snapshot,
дающий стабильно неверный вывод. Flag form хранит нерешённые границы в самом CS
узла, где они остаются под рёбрами.

История нарушения: `Concretest([..]?)` фабриковал `opt(fresh-CS)` — семейство багов
dead-invisible-snapshot (FU710, `StaleSnapshotSyntaxTests`, nested none-joins вида
`if(true) [[1]] else [[none]]`). Закрыто 2026-07: семейство canonical-Optional-form
в TicTechnicalDebt §Closed.

## Descendant flag-form — opt-free

> Внутри flag form ось Optional уже несёт флаг, поэтому хранимый Descendant обязан
> быть opt-свободным: `[opt(X)..A]?` ≡ `opt(opt(X))` — неканонично.

**Правило среза**: при записи Descendant в optional-CS внешняя `opt`-обёртка
снимается:

```
store([D..A]?, opt(X))  →  [X..A]?
```

Без среза интервальные проверки ломаются на ровном месте: `[F32..Real]` против
`[opt(F32)..]` несравнимы, хотя семантически это один и тот же optional-интервал.

## Opt(Any) = Any — фактор-отношение

Из постулатов (`Algebra.md` §Постулаты): `Opt(T) = T | None`, `None ⊆ Any` ⟹
`Opt(Any) = Any | None = Any`.

Домен типов берётся **по модулю** этого отождествления:

- `opt(Any)` никогда не хранится — каждый оператор коллапсирует его в `Any`
  (ветки `Opt(Any) → Any` в ∨, ∧, ↓, ↑, lift).
- Антисимметричность ≤ выполняется именно по модулю: `Any ≤ opt(Any)` (implicit
  lift) и `opt(Any) ≤ Any` (top) — без отождествления это два разных типа,
  нарушающих partial order.

Тесты: `Lca_OptAnyEqualsAny`, `Gcd_OptAnyIsIdentity`,
`Concretest_OptionalAny_CollapsesToAny`, `OptAny_CollapsesToAny`.

## opt(opt(T)) = opt(T) — flatten как реактивное поддержание

Постулат: вложенных Optional не существует (`Opt(Opt(T)) = Opt(T)`). Однако при
**отложенном распространении ограничений** состояние element-узла меняется
асинхронно — уже ПОСЛЕ того, как родительский `opt(...)` установлен, элемент может
сам стать Optional через другое ребро. Предотвратить создание вложенности на
записи нельзя (родитель уже записан); её можно только выправить при следующем
касании узла — поэтому поддержание **реактивное**: `FlattenNestedOptional`,
три точки вызова:

1. **Установка состояния узла** — при присваивании `opt(...)`-состояния элемент
   разыменовывается, и если он уже Optional, внешний слой снимается на месте.
2. **Destruction (рекурсивный обход)** — после обработки composite-членов:
   материализация IsOptional (подфаза 5b) могла вложить `opt` в уже-Optional член.
3. **Finalize** — safety net после дереференса RefTo: сжатие ссылочных цепочек
   может обнажить вложенность, собранную из разных путей графа.

Все три точки — одна и та же операция нормализации к канонической форме; различие
только в моменте, когда узел оказывается «под рукой». Принятый дизайн (см.
CLAUDE.md §Accepted Design): реактивный flatten корректен для deferred propagation.

## Закон refinability

> **Refinability**: каждая нерешённая граница остаётся достижимой по рёбрам графа.
> Ни один оператор алгебры не создаёт unsolved-состояние, не достижимое из узлов,
> участвующих в рёбрах.

До сих пор этот инвариант существовал только как комментарий в коде — здесь он
возведён в закон системы (закон 12 умбреллы, `Algebra.md` §Инварианты). Rule B —
его Optional-инстанс; правило `↓Struct = identity` (запрет покомпонентного
snapshot-а полей, `Algebra_Concretest.md` §Struct) — его Struct-инстанс.

Нарушение = dead island: снапшот с нерешённой границей, который Pull/Push никогда
не уточнит. Симптом всегда один — «замороженный» тип, противоречащий остальному
выводу (FU710-семейство).

Тест-инстанс: `CanonicalOptionalFormTest.Concretest_ArrayOfEmptyOptionalCs_NoDeadOptIsland`
(форма `[none]` внутри `[[none]]` в момент снапшота).

## Замкнутость операторов относительно канонических форм

**Теорема (сохранение каноничности)**: если входы канонические, то результат
каждого оператора канонический.

По операторам:

- **∨ (LCA)**: optional-ветки строят flag form и применяют правило среза к
  Descendant; `Opt(Any)` коллапсируется. Тесты:
  `Lca_ArrayIntPref_vs_ArrayEmptyOptional_KeepsIntervalAndHint`,
  `Lca_NestedArrays_ReducesToFlatByInduction`.
- **∧ (GCD)**: optional-CS разворачивается до примитивной верхней границы —
  результат либо решённый тип, либо null; вложенность не создаётся.
- **⊓ (Merge)**: транспорт `opt = opt₁ ∨ opt₂` остаётся на флаге; коллапс в
  `opt(T)` происходит только при решённом точечном интервале.
- **↓ (Concretest)**: Rule B ветки — единственное место алгебры, где флаг
  материализуется в конструктор, и только над решённым inner.
- **↑ (Abstractest)**: сбрасывает ось opt (парный контракт с Destruction,
  `Algebra_Abstractest.md`) — результат-примитив тривиально каноничен; cmp-ветка
  возвращает CS с флагом (flag form канонична).
- **simp (Simplify)**: валидирует интервал с учётом флага (`opt(T)` — композит, не
  удовлетворяет примитивному Ancestor ≠ Any) и материализует только решённые точки.

Верификация: `CanonicalOptionalFormTest` (TIC-уровень),
`StaleSnapshotSyntaxTests` (syntax-уровень, историческое семейство нарушений).
