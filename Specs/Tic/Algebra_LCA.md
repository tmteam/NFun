# LCA — Lowest Common Ancestor

## Определение

**LCA(A, B)** — наименьший (самый конкретный) тип, который является супертипом и A, и B.

Обозначение: **A ∨ B** (join в решётке типов).

## Тотальность

LCA определён для **любой** пары типов. `Any` — top решётки (∀T: T ≤ Any), поэтому A ∨ B всегда существует. В худшем случае A ∨ B = Any.

## Алгебраические свойства

| Свойство | Формула | Статус |
|---|---|---|
| Идемпотентность | `A ∨ A = A` | Тривиально |
| Коммутативность | `A ∨ B = B ∨ A` | Тривиально |
| Ассоциативность | `(A ∨ B) ∨ C = A ∨ (B ∨ C)` | Примитивы: верифицировано таблицей. Composites: структурная индукция |
| Монотонность | `A ≤ B ⟹ A ∨ C ≤ B ∨ C` | Из определения join |
| Поглощение | `A ∨ Any = Any` | Any — top |
| Ref-прозрачность | `Ref(A) ∨ B = A ∨ B` | Ссылки разыменовываются до вычисления |

## Инварианты

1. **Тотальность**: ∀A,B: A ∨ B определён
2. **Верхняя грань**: A ≤ A ∨ B, B ≤ A ∨ B
3. **Наименьшая верхняя грань**: A ≤ C ∧ B ≤ C ⟹ A ∨ B ≤ C
4. Свойства 4-6 из таблицы выше (идемпотентность, коммутативность, ассоциативность)
5. **None lift**: None ∨ T = Opt(T) для T ∉ {Any, None}
6. **Ref-прозрачность**: Ref(A) ∨ B = A ∨ B

## Правила по категориям типов

### Примитивы

Примитивные типы образуют частичный порядок (решётку) с Any наверху. Если два примитива имеют общего предка конкретнее Any — LCA возвращает его. Иначе — Any.

Например: `I32 ∨ I64 = I64`, `I16 ∨ U16 = I24`, `Bool ∨ I32 = Any`.

Реализация: предвычисленная таблица, O(1). Конкретная иерархия — см. `Types.md`.

### None

None — bottom для optional. LCA с None поднимает тип в Optional:

```
None ∨ None   = None
None ∨ T      = Opt(T)      для T ∉ {Any, None}
None ∨ Any    = Any          (Opt(Any) = Any)
None ∨ Opt(T) = Opt(T)
```

### Составные (composite) типы — общее правило

Для составного типа `F<T₁, T₂, ...>`, LCA вычисляется **покомпонентно** с учётом вариантности:

| Вариантность | Компонента результата | Оператор |
|---|---|---|
| **Ковариантная** (out) | `Tᵢ' = Tᵢ_a ∨ Tᵢ_b` | LCA |
| **Контравариантная** (in) | `Tᵢ' = Tᵢ_a ∧ Tᵢ_b` | GCD |

Если типы **структурно несовместимы** (разные конструкторы), результат = Any:

```
F<...> ∨ G<...> = Any      (если F ≠ G)
```

Примеры:

- **Array** `T[]` — ковариантен по T: `A[] ∨ B[] = (A ∨ B)[]`
- **Optional** `Opt(T)` — ковариантен по T: `Opt(A) ∨ Opt(B) = Opt(A ∨ B)`, `Opt(A) ∨ B = Opt(A ∨ B)`
- **Function** `(T₁,T₂)→R` — контравариантна по аргументам, ковариантна по возврату: `(A→R₁) ∨ (B→R₂) = (A ∧ B) → (R₁ ∨ R₂)`. Если argsCount различен — Any.

### Struct — width subtyping

Struct — особый composite. LCA = **пересечение имён полей** с ковариантными типами:

```
{x:A, y:B} ∨ {x:C, z:D} = {x: A ∨ C}
{x:A}       ∨ {y:B}      = {}
```

Поле исключается если отсутствует в одном из struct или LCA типов поля не определён.

Рекурсивные структуры: LCA(T, T) = T при обнаружении цикла.

## ConstraintsState

ConstraintsState (CS) — нерешённый тип, представленный интервалом `[Desc..Anc]` и флагами:
- **Desc** — нижняя граница (наиболее конкретный допустимый тип). Может быть null (не ограничен снизу).
- **Anc** — верхняя граница (наиболее общий допустимый тип). Может быть null (не ограничен сверху).
- **IsOptional** — тип может быть None (из ветки if-else с none).
- **IsComparable** — тип должен поддерживать сравнение (constraint, не свойство).
- **StructBound (S)** — опциональная структурная верхняя граница (F-bounded). См. PushReform.md.
- **Preferred** — hint для resolution (например, I32 для целочисленных литералов). **Не участвует в LCA.**

LCA для CS — по трём путям:

**CS ∨ CS:**
```
desc = D₁ ∨ D₂       (расширяет снизу; если один пуст — Concretest другого)
anc  = ∅              (ancestor ТЕРЯЕТСЯ — join расширяет, не сужает сверху)
opt  = opt₁ OR opt₂   (любой optional → результат optional)
cmp  = cmp₁ AND cmp₂  (constraint сужается при join, не расширяется)
S    = LcaStruct(S₁,S₂); null absorbs (если одна сторона без bound — bound теряется)
```

`S` ведёт себя как ancestor: LCA — join, расширяет; null absorbs потому что отсутствие bound = "нет верхнего структурного ограничения" — самый широкий вариант. `LcaStruct` — пересечение полей с рекурсивным LCA на общих полях (как `StateStruct ∨ StateStruct` в существующем правиле).

Почему ancestor теряется: LCA — join (расширение). Ancestor — верхняя граница (сужение). Join не добавляет верхних ограничений. GCD и Unify сужают ancestor; LCA — нет.

IsComparable — AND (не OR): comparable это **ограничение** (сужает допустимые типы). LCA ослабляет ограничения.

IsOptional — OR: optional это **расширение** (добавляет None в допустимые значения). LCA сохраняет расширения.

Preferred **не участвует** в LCA. Preferred — hint для финального выбора типа, не для алгебраических операций. Описан в разделе ConstraintsState как часть состояния узла, но алгебраически инертен.

**ConcreteType ∨ CS:** `T ∨ [D..] = T ∨ ↓D`. Если CS.IsOptional и результат ≠ Any → результат оборачивается в CS с IsOptional=true.

Симметричен: CS ∨ T = T ∨ CS.

## Связь с другими операторами

- **GCD** (`∧`) — дуальный. Может быть null. См. `Algebra_GCD.md`.
- **FitsInto** — `A ≤ B` ⟺ `A ∨ B = B`
- **Unify** — строже LCA, может вернуть null. Используется в struct LCA для CS-полей.

## Сложность

| Входные типы | Сложность |
|---|---|
| Primitive × Primitive | O(1) |
| Composite × Composite | O(components × depth) |
| Recursive struct | + O(nesting) cycle guard |
