# GCD — Greatest Common Descendant

## Определение

**GCD(A, B)** — наибольший (самый общий) тип, который является подтипом и A, и B.

Обозначение: **A ∧ B** (meet в решётке типов).

Интуиция: "наиболее широкий тип, значения которого принимаются и там где ожидается A, и там где ожидается B".

## Частичность

В отличие от LCA, GCD **может не существовать** (возвращает null). Два типа могут не иметь общего подтипа.

Например: `Bool ∧ I32 = null` — нет типа, который одновременно Bool и I32.

Any — top: `A ∧ Any = A` (Any принимает всё, ограничение не добавляется).

## Алгебраические свойства

| Свойство | Формула | Статус |
|---|---|---|
| Идемпотентность | `A ∧ A = A` | Тривиально |
| Коммутативность | `A ∧ B = B ∧ A` | Тривиально |
| Ассоциативность | `(A ∧ B) ∧ C = A ∧ (B ∧ C)` | Для примитивов: верифицировано таблицей. Composites: структурная индукция |
| Монотонность | `A ≤ B ⟹ A ∧ C ≤ B ∧ C` | Из определения meet (где обе стороны определены) |
| Top | `A ∧ Any = A` | Any не сужает |
| Null-propagation | `A ∧ B = null ⟹ ∀C≤A: C ∧ B = null` | Если нет общего подтипа — подтипы тоже не помогут |
| Ref-прозрачность | `Ref(A) ∧ B = A ∧ B` | Ссылки разыменовываются |

## Дуальность с LCA

GCD — дуальный оператор к LCA:

| LCA (∨) | GCD (∧) |
|---|---|
| Наименьший **супер**тип | Наибольший **под**тип |
| Всегда определён | Может быть null |
| `A ∨ Any = Any` | `A ∧ Any = A` |
| Join (расширяет) | Meet (сужает) |

Связь: `A ≤ B` ⟺ `A ∨ B = B` ⟺ `A ∧ B = A`

## Правила по категориям типов

### Примитивы

Если два примитива имеют общий подтип — GCD возвращает его. Иначе — null.

Например: `I64 ∧ Real = I64` (I64 ≤ Real), `I32 ∧ U32 = U24` (перекрёстный), `Bool ∧ I32 = null`.

Реализация: предвычисленная таблица 19×19, O(1). Null-записи = нет общего подтипа.

### None

None — bottom для optional типов, но НЕ bottom для всех типов:

```
None ∧ None   = None
None ∧ Any    = None         (None ≤ Any)
None ∧ Opt(T) = None         (None ≤ Opt(T))
None ∧ T      = null         (None ≰ конкретным не-optional типам)
```

### Составные (composite) типы — общее правило

Дуально к LCA. Для `F<T₁, T₂, ...>`:

| Вариантность | Компонента результата | Оператор |
|---|---|---|
| **Ковариантная** (out) | `Tᵢ' = Tᵢ_a ∧ Tᵢ_b` | GCD |
| **Контравариантная** (in) | `Tᵢ' = Tᵢ_a ∨ Tᵢ_b` | LCA |

Если типы структурно несовместимы (разные конструкторы) — null:

```
F<...> ∧ G<...> = null      (если F ≠ G)
```

Если GCD любой компоненты = null → весь GCD = null.

Примеры:

- **Array**: `A[] ∧ B[] = (A ∧ B)[]`. Если `A ∧ B = null` → null.
- **Optional**:
  - `Opt(A) ∧ Opt(B) = Opt(A ∧ B)` — ковариантно по inner
  - `Opt(A) ∧ Any = Opt(A)` — Opt(A) ≤ Any
  - `Opt(A) ∧ B = A ∧ B` — для B ∉ {Any, Optional}. Почему: если C ≤ Opt(A) и C ≤ B, то C ≠ None (т.к. None ≰ B), значит C ≤ A. Поэтому meet = meet без обёртки.
- **Function**: `(A→R₁) ∧ (B→R₂) = (A ∨ B) → (R₁ ∧ R₂)`. Аргументы — LCA (контравариантно), возврат — GCD (ковариантно). Если argsCount различен — null.

### Struct — width subtyping

Дуально к LCA: struct GCD = **объединение полей** (union). Больше полей = более конкретный тип:

```
{x:A, y:B} ∧ {x:C, z:D} = {x: A ∧ C, y:B, z:D}
{x:A}       ∧ {y:B}      = {x:A, y:B}
```

Для общих полей — GCD типов. Если GCD общего поля = null → весь struct GCD = null.

## ConstraintsState

**CS ∧ ConcreteType:** CS раскрывается через **верхнюю** границу (Ancestor):
```
[D..A] ∧ T = A ∧ T              (Ancestor определён — сужаем по нему)
[D..∅] ∧ T = Abstractest(T)     (нет Ancestor — возвращаем T как есть)
```

`Abstractest(T)` — наиболее абстрактная форма типа. Для конкретных типов = identity. Для composites — рекурсивное обобщение элементов.

**CS ∧ CS:** аналогично — оба раскрываются через Ancestor.

**StructBound (S):** `GcdStruct(S₁, S₂)` — null absorbs к другому; если оба non-null — **union полей** с рекурсивным GCD на общих. GCD сужает (richer bound), дуально к LCA. См. PushReform.md.

**Desc, IsOptional, IsComparable** — при раскрытии CS в GCD теряются (используется только Ancestor). Это корректно: GCD сужает тип, а Desc/flags — нижняя граница и constraints, не верхняя. `StructBound` сохраняется потому что это **третья независимая верхняя граница** (структурная), а не lower-bound constraint.

**Preferred** не участвует (resolution hint).

## Граничные случаи

| Случай | Результат | Почему |
|---|---|---|
| `A ∧ A` | `A` | Идемпотентность |
| `A ∧ Any` | `A` | Any не сужает |
| `Bool ∧ I32` | `null` | Нет общего подтипа |
| `A[] ∧ {x:B}` | `null` | Разные конструкторы |
| `None ∧ I32` | `null` | None ≰ I32 |
| `None ∧ Opt(I32)` | `None` | None ≤ Opt(T) |

## Связь с другими операторами

- **LCA** (`∨`) — дуальный. Всегда определён. См. `Algebra_LCA.md`.
- **FitsInto** — `A ≤ B` ⟺ `A ∧ B = A`
- **TryAddAncestor** — сужение верхней границы CS: `Anc' = Anc ∧ new_anc`. Если null — интервал пуст (ошибка типа).

## Сложность

Аналогична LCA:

| Входные типы | Сложность |
|---|---|
| Primitive × Primitive | O(1) |
| Composite × Composite | O(components × depth) |

## Инварианты

1. **Нижняя грань** (если определена): A ∧ B ≤ A, A ∧ B ≤ B
2. **Наибольшая нижняя грань**: C ≤ A ∧ C ≤ B ⟹ C ≤ A ∧ B
3. **Null-propagation**: A ∧ B = null ⟹ ¬∃T: T ≤ A ∧ T ≤ B
4. Свойства из таблицы: идемпотентность, коммутативность, ассоциативность, top, ref-прозрачность
